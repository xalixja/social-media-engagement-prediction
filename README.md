## Project Goal

The main goal of this project is to predict the **engagement level** of social media posts based on tweet content, using Natural Language Processing (NLP) and classical machine learning techniques.

Engagement is operationalized as the number of retweets, which is bucketed into three classes — **Low**, **Medium**, and **High** — based on quantile thresholds of the retweet distribution. The project investigates two main questions:

1. Can the words, phrases, and stylistic features of a tweet reliably predict its engagement class?
2. Which elements of a tweet most strongly drive the model's decision toward a given class?

The second question is answered through an explainability analysis using **SHAP (SHapley Additive exPlanations)**, applied to the final classification model.

## Methodology

### 1. Exploratory Data Analysis (`01_EDA.ipynb`)
Comparison of the two source CSVs (`realdonaldtrump.csv` vs. `trumptweets.csv`), selection of the more complete file, analysis of the retweet distribution, definition of the three engagement classes via quantile thresholds, temporal trends in engagement, and early Pearson correlation checks between retweets, tweet length, and favorites.

### 2. Preprocessing (`02_preprocessing.ipynb`)
Lowercasing, URL and media-link removal, retention of digits (years, vote counts, etc.), conversion of mentions and hashtags into dedicated tokens (`mentions_<name>`, `hashtag_<name>`) instead of stripping them, tokenization, selective stopword removal (personal pronouns, negations, and modal verbs are kept), and lemmatization via `WordNetLemmatizer`. A word cloud and top-token frequency plot summarize the cleaned vocabulary.

### 3. Feature Engineering (`03_features.ipynb`)
- **45 handcrafted numeric features**, including punctuation counts, capitalization ("shouting") indices, pronoun and rhetorical-device counts, readability scores (Flesch Reading Ease, Flesch–Kincaid Grade), lexical diversity (TTR), and numeric-token flags (e.g. year mentions vs. other large numbers).
- **Sentiment analysis** via **VADER**, producing negative/neutral/positive/compound scores.
- **Text vectorization** via `CountVectorizer` and `TF-IDF` (unigrams + bigrams, vocabulary capped at 10,000 terms).
- **Topic modeling** via `LatentDirichletAllocation` (5 topics) fitted on the Bag-of-Words representation.
- All numeric features are standardized with `StandardScaler` and combined with the sparse text matrices into two final feature matrices (`X_tfidf_features.npz`, `X_count_features.npz`), each of shape `(43352, 10045)`.

### 4. Modeling (`04_models.ipynb`)
A stratified 80/20 train-test split, followed by a benchmark of seven model/representation combinations:

| Model | Text representation |
|---|---|
| `DummyClassifier` (most frequent) | — |
| Logistic Regression | TF-IDF + numeric |
| Linear SVM | TF-IDF + numeric |
| Logistic Regression | Count + numeric |
| Linear SVM | Count + numeric |
| XGBoost | TF-IDF + numeric |
| XGBoost | Count + numeric |

Models are compared on accuracy, balanced accuracy, macro F1, and weighted F1, with **macro F1** as the primary selection criterion. The best-performing model is serialized to `models/best_engagement_model.joblib`.

### 5. Explainability (`05_shap.ipynb`)
SHAP values are computed with `shap.LinearExplainer` for the selected (linear) model, over a sample of test tweets. The notebook produces global feature importance rankings, per-class importance and signed-direction plots, a breakdown of engineered-feature vs. text-n-gram contribution, and local explanations for individual example tweets.

## Dataset

This project uses the **Trump Tweets dataset** available on Kaggle:

[https://www.kaggle.com/datasets/austinreese/trump-tweets](https://www.kaggle.com/datasets/austinreese/trump-tweets)

The dataset contains tweets published by Donald Trump along with metadata such as retweet counts, favorites, timestamps, and tweet sources.

### Download instructions

1. Go to the Kaggle dataset page linked above.
2. Download the dataset as a `.zip` file.
3. Extract the files locally.

### Required file structure

After extraction, place the dataset files into the following directory:

```
data/
└── raw/
    ├── realdonaldtrump.csv
    └── trumptweets.csv
```

### Note

The raw dataset is **not included** in this repository due to file size constraints and reproducibility best practices. Each user must download it independently from Kaggle. All intermediate and processed artifacts (cleaned text, feature matrices, trained model) are generated locally by running the notebooks in order.

## Technologies and Tools

- **Data handling:** Pandas, NumPy, SciPy (sparse matrices)
- **NLP:** NLTK (tokenization, stopwords, `WordNetLemmatizer`, VADER sentiment), `textstat` (readability scores), `wordcloud`
- **Feature extraction:** Scikit-learn (`CountVectorizer`, `TfidfVectorizer`, `LatentDirichletAllocation`, `StandardScaler`, `LabelEncoder`)
- **Modeling:** Scikit-learn (`LogisticRegression`, `LinearSVC`, `DummyClassifier`), XGBoost
- **Explainability:** SHAP
- **Visualization:** Matplotlib, Seaborn
- **Model persistence:** joblib
- **Environment:** Jupyter Notebook

## Setup

```bash
uv venv
source .venv/bin/activate
uv pip install -r requirements.txt
```

The first time NLTK-based notebooks are run, the required corpora (stopwords, WordNet, VADER lexicon) may need to be downloaded via `nltk.download(...)` — this is handled inside the notebooks themselves.

## Running the Project

1. Download the dataset and place the raw CSVs in `data/raw/` (see [Dataset](#dataset)).
2. Run the notebooks **in order** from the `notebooks/` directory:
   ```
   notebooks/01_EDA.ipynb
   notebooks/02_preprocessing.ipynb
   notebooks/03_features.ipynb
   notebooks/04_models.ipynb
   notebooks/05_shap.ipynb
   ```
3. Each notebook reads the artifacts produced by the previous one from `data/processed/` and writes its own outputs there (or to `models/` for the trained model).

### Jupyter / PyCharm Configuration

If notebook imports (e.g. `pandas`) do not work, ensure that:

- the correct `.venv` interpreter is selected in PyCharm or Jupyter.

## LaTeX Report Environment (VS Code, Linux)

The project report (`docs/main.tex`) is written in LaTeX. Below is a short guide for setting up a Linux system and VS Code to build it.

### 1. Install a LaTeX distribution

```bash
sudo apt update
sudo apt install texlive-full
sudo apt install python3-pygments
```

`python3-pygments` is required because the report uses the `minted` package for syntax-highlighted code listings.

### 2. VS Code extension

1. Open VS Code.
2. Go to the Extensions tab (`Ctrl + Shift + X`).
3. Search for and install **LaTeX Workshop** (by *James Yu*).

### 3. Enable `-shell-escape` for `minted`

By default, LaTeX disallows running external programs (such as Pygments) for security reasons. This must be explicitly enabled via the `-shell-escape` flag:

1. Press `Ctrl + Shift + P` to open the command palette.
2. Type: `Preferences: Open User Settings (JSON)`.
3. Add the following:

```json
"latex-workshop.latex.tools": [
    {
        "name": "latexmk",
        "command": "latexmk",
        "args": [
            "-synctex=1",
            "-interaction=nonstopmode",
            "-file-line-error",
            "-pdf",
            "-outdir=%OUTDIR%",
            "-shell-escape",
            "%DOC%"
        ]
    },
    {
        "name": "pdflatex",
        "command": "pdflatex",
        "args": [
            "-synctex=1",
            "-interaction=nonstopmode",
            "-file-line-error",
            "-shell-escape",
            "%DOC%"
        ]
    }
]
```

### 4. Working with the report

- **Compilation:** the extension recompiles the PDF automatically on every save (`Ctrl + S`).
- **PDF preview:** open the live preview next to the editor with `Ctrl + Alt + V`, or via the preview icon in the top-right corner.
- **Navigation:** `Ctrl + Click` on text in the PDF preview jumps to the corresponding line in the `.tex` source, and vice versa (SyncTeX).
